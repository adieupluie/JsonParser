# JSON Parser Viewer UI - 탭 기반 디자인

## 1. 탭 기반 화면 전환 제안

기존 3분할 레이아웃을 탭 방식으로 바꾸면 다음과 같은 장점이 있습니다:

- 작업 공간이 넓어지고 단일 화면에 집중 가능
- 입력, 트리 탐색, 상세 정보가 각각 독립된 탭으로 분리
- 사용자가 필요할 때만 해당 탭을 열어 UI 복잡도 감소

## 2. 탭 구성

### 탭 1: JSON 입력

- 제목: `JSON 입력`
- 내용:
  - 멀티라인 텍스트 에디터
  - 라인 번호, 구문 하이라이트
  - `Parse on Type` 토글
  - `Parse` 버튼
  - 오류 메시지와 `Go to Error` 링크

### 탭 2: 트리 뷰

- 제목: `Tree View`
- 내용:
  - JSON 파싱 결과를 계층 트리로 표시
  - 노드 선택/확장/축소 기능
  - 우클릭 메뉴: `Copy Path`, `Copy Value`, `Focus in Editor`
  - 검색 필드(노드/값 검색)

### 탭 3: 노드 인스펙터

- 제목: `Inspector`
- 내용:
  - 선택 노드의 `path`, `type`, `value` 표시
  - 값 편집 입력란
  - `Apply` 버튼
  - 기타 메타 정보/주석

## 3. 화면 흐름

1. 사용자가 `JSON 입력` 탭에서 텍스트를 입력하고 파싱.
2. 파싱 성공 시 `Tree View` 탭에서 결과 확인.
3. 노드를 선택하면 `Inspector` 탭에서 상세 정보 확인.
4. 필요 시 `JSON 입력` 탭으로 돌아가 텍스트 수정.

## 4. 탭 전환 UX

- 기본 포커스는 `JSON 입력` 탭
- 파싱 성공 후 `Tree View`로 자동 이동 옵션
- 노드 선택 시 `Inspector` 탭을 활성화하는 버튼
- 탭 제목에 상태 배지 표시 가능(`오류`, `완료`)

## 5. 장단점

### 장점
- 단순하고 집중된 화면 구성
- 작은 화면에서도 UI가 더 깔끔함
- 각 모드별 기능을 명확히 분리

### 단점
- 사용자가 탭을 전환해야 해서 단계가 늘어남
- 실시간 트리와 에디터 동시 비교가 어려움

## 6. 추천 조합

- 기본: `JSON 입력` 탭 + `Tree View` 탭
- 고급: `Inspector`는 선택 노드가 있을 때만 활성화
- `Tree View`에서 `Open in Editor` 기능으로 `JSON 입력`으로 곧바로 이동
